Entry Point
main.py
FastAPI app
Registers routers, CORS middleware, exception handlers, startup/shutdown lifecycle. Serves static frontend if present.
dependencies.py
ConnectionStateManager
Per-session state keyed by session_id:tab_id. Provides get_state_manager, get_active_connector, get_connection_details as FastAPI dependencies.
exceptions.py
Error handling
AppException, DataSourceConnectionError, QueryExecutionError. Mapped to HTTP status codes by main.py handlers.
API Routers  /api/v1/data
connection
POST /connect, /disconnect
Connect/disconnect. File uploads (multipart), JSON connect, Hive Parquet two-phase flow, add-files.
metadata
GET /databases, /tables, /columns
List databases, tables, and columns from the active connector.
query
POST /query, /query-arrow
Execute queries (JSON or Arrow IPC), row count, distinct count.
relationships
GET /table-relationships
Foreign key detection, suggested joins, merged columns for multi-table queries.
snapshot
GET/POST /snapshots
Snapshot CRUD – save/load query configurations to file-based JSON storage.
kaggle
GET /kaggle/search
Search Kaggle datasets, list files for import.
Services
ConnectionService
services/connection_service.py
Orchestrates connect/disconnect. Parses ConnectionDetails, saves uploads, creates connector via ConnectorRegistry, sets session state.
ValidationService
services/validation_service.py
Validates ClickHouse DB existence, CSV table names, and query fields before execution.
TableMergeService
services/table_merge_service.py
Builds virtual tables from suggested joins. Detects foreign keys, computes merged columns.
SnapshotService
services/snapshot_service.py
File-based JSON snapshot storage. CRUD operations on saved query configurations.
KaggleSearchService
services/kaggle_search_service.py
Wraps Kaggle API – search datasets, list files, trigger download for import.
CardinalityService
services/cardinality_service.py
Computes distinct counts for columns. Used by UI for filter dropdowns and optimization hints.
Query Pipeline
QueryExecutionService
Orchestrator
parse → validate → translate → execute → build result. Central query pipeline coordinator.
QueryService
SQL Generation (PyPika)
Translates QueryDescription → SQL via PyPika. Coordinates all query component builders.
QueryResultBuilder
Result formatting
Transforms raw connector output (columns + rows) into QueryResult with metadata, types, JS-safe numbers.
DateTimeService
Date/time handling
Datetime extraction and truncation per DB engine (ClickHouse vs DuckDB). EXTRACT / DATE_TRUNC.
FilterConversionService
Filter translation
Converts frontend filter configuration (UI format) into internal Filter objects for SQL generation.
Query Components  (services/query_components/)
TableContextBuilder
PyPika table context – JOINs, UNIONs, virtual tables, aliases.
SelectClauseBuilder
SELECT clause: dimensions, measures (SUM, AVG, COUNT…), virtual columns.
FilterBuilder
WHERE clause. Operators: eq, ne, gt, lt, in, between, like, is_null.
GroupingOrderingBuilder
GROUP BY and ORDER BY generation. Dimension grouping, multi-column ordering.
OptimizationApplier
Applies optimization strategies: adaptive rounding, category dedup, datetime binning.
ResultBudgetApplier
Sampling & LIMIT for result budgets. TABLESAMPLE (CH), row limits (DuckDB).
VirtualColumnBuilder
Virtual column expressions for computed fields.
UnionQueryBuilder
UNION query construction with null columns, source filters, virtual column checks.
Optimization Engine  (services/optimization/)
QueryOptimizer
Coordinates: config → detect small tables → plan strategies → estimate cardinality → select best plan.
StrategyPlanner
Builds strategies from hints or query structure. Adaptive rounding, category dedup, datetime binning.
SmallTableDetector
Checks row count vs threshold. Skips optimization for small tables.
OptimizerConfig
Configuration from env vars: thresholds, cache settings, strategy toggles.
ClickHouseEstimator
system.parts + uniqExact for cardinality estimation.
DuckDBEstimator
approx_count_distinct for cardinality estimation.
BasicEstimator
Fallback estimator when engine-specific estimators aren't available.
Data Connectors
ConnectorRegistry
connectors/registry.py
Factory mapping connection type strings to connector classes. create(type) → BaseConnector subclass.
BaseConnector (ABC)
connectors/base.py
Abstract interface: connect, disconnect, list_databases, list_tables, list_columns, fetch_data, fetch_data_arrow.
Concrete Connectors  (implement BaseConnector)
ClickHouseConnector
clickhouse-connect
HTTP client with thread-safe lock. Detects foreign keys via system tables. Supports Arrow output.
FileConnector
DuckDB (in-memory)
CSV/Parquet via DuckDB. File handlers for validation and schema inference. Per-session temp files.
KaggleConnector
Kaggle API + DuckDB
Downloads Kaggle datasets, then loads into DuckDB. Extends FileConnector pattern.
HiveParquetConnector
DuckDB (partitioned)
Two-phase: (1) connect to discover partitions, (2) load selected partition. DuckDB backend.
File Handlers  (connectors/file_handlers/)
CsvFileHandler
CSV validation and schema inference.
ParquetFileHandler
Parquet validation and schema inference.
Shared Infrastructure
Models (Pydantic)
models/
QueryDescription, QueryResult, ConnectionDetails, DataSource, Column, Filter, Measure, Dimension, and more.
Utils
utils/type_conversion.py
Decimal→float, NaN/Inf→None, JS-safe integer checks. Used by QueryResultBuilder.
Exceptions
exceptions.py
AppException, DataSourceConnectionError, QueryExecutionError. Mapped to HTTP codes by main.py.
CORS Middleware
via main.py
CORSMiddleware configured from CORS_ALLOW_ORIGINS env var.
Config / Env
Environment variables
LOG_LEVEL, CORS_ALLOW_ORIGINS, SNAPSHOT_STORAGE_DIR, OPTIMIZER_* thresholds and cache settings.

Query flow: POST /query → Router → QueryExecutionService → validate → QueryService (SQL) → Connector.fetch_data → QueryResultBuilder → JSON

Connect flow: POST /connect → Router → ConnectionService → ConnectorRegistry.create(type) → connector.connect() → StateManager.set_state()
